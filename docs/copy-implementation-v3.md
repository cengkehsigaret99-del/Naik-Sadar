# Copy Implementation V3

Dokumen ini menjadi acuan implementasi teks UI Modul Kebersihan.

## Prinsip

- Bahasa layar harus umum dan objektif.
- Teks harus menjelaskan tugas pengguna.
- Tombol memakai kata kerja langsung.
- Pilihan jawaban memakai kondisi area atau hasil tindakan.
- Kerangka filosofi tetap di dokumen internal, bukan instruksi utama UI.

## Struktur Layar

1. Home: jelaskan fungsi modul.
2. Pilih Area: minta pengguna memilih satu area.
3. Cek Awal: minta pengguna menilai kondisi area.
4. Jeda Napas: beri jeda singkat sebelum tindakan.
5. Tindakan: minta pengguna memilih satu tindakan kecil.
6. Evaluasi Sesudah: catat hasil tindakan.
7. Catatan: tampilkan riwayat sesi.

## Copy Final

Home:
Evaluasi kondisi area, pilih tindakan perapian singkat, dan catat hasil setelah tindakan.

Pilih Area:
Area mana yang akan dievaluasi?

Cek Awal:
Bagaimana kondisi area saat ini?

Jeda Napas:
Ambil jeda singkat sebelum mulai.

Tindakan:
Pilih satu tindakan kecil.

Evaluasi Sesudah:
Apa hasil setelah tindakan?

Catatan:
Riwayat Modul Kebersihan.

## Pilihan Kondisi

- Terkendali
- Cukup rapi
- Perlu dirapikan
- Mengganggu fokus
- Sering tertunda

## Pilihan Hasil

- Area lebih rapi
- Area lebih mudah digunakan
- Akses lebih lancar
- Masih perlu dilanjutkan
- Cukup untuk hari ini
- Perlu evaluasi ulang
